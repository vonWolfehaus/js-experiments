Endless pages

Overview

Inspired by the Humanized Reader (see their excellent post for the rationale), I've implemented the concept with jQuery in this very basic prototype (it just loads a static html page, but it could easily be adapted to fetch an RSS feed for example). Simply scroll to the bottom of the page and the new content will be loaded automatically.

What sets their implementation apart is their attention to detail. The user only needs to know what's happening if their actions require it. So as long as they scroll down using the scroll wheel, they won't notice a thing -- except an extraordinarily long page ;) But because of the way the browser works, loading new content via AJAX can be jarring. In other words, when something gets loaded into the DOM and the user is currently scrolling using the scrollbar, their window will jump somewhere else since the length of the page changed (the scrollbar operates on percentages).

So to avoid that problem the user is informed that new content is being loaded and will load as long as they keep off the scrollbar until it's complete. That way no jumping occurs and the user won't be jarred.

However instead of simply removing the information bar, I turned it into a navigational aid. With it, users can more easily transverse the document with speed yet know exactly what page they're at with a glance. This allows users to mentally bookmark certain areas of content, the same function pages used to provide.

This is ... it's time we stop demanding that users do work that the computer could do itself.
